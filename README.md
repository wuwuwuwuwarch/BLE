# 蓝牙室内定位系统 (BLE Indoor Positioning System)

![Android](https://img.shields.io/badge/Android-34-green) ![BLE](https://img.shields.io/badge/BLE-4.0+-blue) ![License](https://img.shields.io/badge/License-MIT-orange)

## 项目简介

本项目是一个基于 Android 平台的蓝牙低功耗（BLE）室内定位系统，版本号为 2.0。该系统利用蓝牙信标的 RSSI（接收信号强度指示）信号，结合多种滤波算法和传感器融合技术，实现室内环境中的设备发现、单点测距和多点定位导航功能。该项目适用于需要进行室内定位研究与开发的场景，如商场导览、仓库物资定位、展馆导航等领域。

## 功能模块

系统采用底部三标签页架构，每个标签页对应一个独立的功能模块：

### 模块一：蓝牙设备搜索（第一页）

该模块负责扫描和发现周围环境中的蓝牙设备，支持两种扫描模式：

- **BLE 扫描模式**：基于 Android 的 BluetoothLeScanner API，采用低延迟扫描模式（SCAN_MODE_LOW_LATENCY），能够快速发现周围的低功耗蓝牙信标设备，实时显示设备名称和信号强度。
- **经典蓝牙扫描模式**：基于 BluetoothAdapter.startDiscovery() 方法，通过系统广播接收器接收设备发现结果，用于发现传统蓝牙设备。

扫描到的设备会实时显示在列表中，每种设备均提供名称、MAC 地址和 RSSI 信号强度等信息。切换标签页时系统会自动停止扫描，避免资源浪费。

### 模块二：单点定位（第二页）

该模块针对单个已知 BLE 信标设备进行距离估算，主要功能包括：

- 输入目标设备的名称（如 Beacon_A）
- 配置 1 米处参考 RSSI 值（RSSI 校准参数，反映距离信标 1 米时的基准信号强度）和环境衰减因子 n（信号传播系数，反映信号在特定环境中的衰减速度）
- 启动定位后自动扫描指定设备并实时显示滤波后的信号强度和估算距离

该模块采用以下技术方案：

**卡尔曼滤波器**：对原始 RSSI 值进行实时滤波处理，有效抑制信号抖动。卡尔曼滤波器的参数设置为过程噪声协方差 Q = 0.01、测量噪声协方差 R = 2.0，能够在快速响应和平滑输出之间取得较好平衡。

**RSSI 距离转换模型**：基于经典的无线信号路径损耗公式，即距离 = 10^((|RSSI| - A) / (10 * n))，其中 A 为 1 米处参考 RSSI 绝对值，n 为环境衰减因子。该方法能够在 1 至 10 米范围内提供较为可靠的距离估算。

**扫描超时机制**：当扫描开启后 10 秒内未发现目标设备，界面会给出橙色提示信息，引导用户确认设备是否已开启、名称是否正确或在搜索页面先行确认设备的存在。

### 模块三：多点定位导航（第三页）

该模块是整个系统的核心功能，实现了基于三个 BLE 信标的室内定位与导航。其工作流程分为以下几个阶段：

**第一阶段：设备扫描与信标配置**

用户点击扫描按钮后发现周围的 BLE 设备，系统会自动显示所有发现的设备名称。用户为三个信标槽位分别指定对应的物理设备，系统支持自动匹配功能。配置完成后，用户可以独立校准每个信标的 1 米 RSSI 值和 n 衰减因子，并通过保存按钮将参数持久化到 SharedPreferences 中。

**第二阶段：采样点采集与信标坐标解算**

用户确定原点位置并点击"记录原点"，然后依次移动至多个位置并记录采样点。在每个采样点处，系统会自动采集来自三个信标的 RSSI 信号，经过卡尔曼滤波后转换为距离值。当某个采样点收集到足够数量的信标距离数据后，系统使用**迭代重加权最小二乘法（IRLS）**解算每个信标的相对坐标位置。

采样点的坐标通过步进距离和手机朝向（基于传感器获得的方位角）计算得出，形成了一个相对的坐标系。当三个信标的坐标均成功解算后，系统会在界面上显示绿色提示"所有信标坐标已解算"。

**第三阶段：导航模式**

用户点击"开始导航"按钮后，系统进入实时导航模式。在该模式下，系统持续扫描三个信标的 RSSI 信号，并通过**三边定位算法（Trilateration）**实时计算手机在坐标系中的当前位置。

定位结果会施加多重平滑约束：

- 位置跳变限制：单次位置变化不超过 1.0 米
- 静止状态检测：当移动距离小于 0.3 米时采用强平滑因子
- 异常值抑制：综合使用指数平滑和卡尔曼滤波

同时，罗盘视图会实时显示指向目标信标的方向和距离，坐标系视图直观展示手机、信标和采样点之间的相对位置关系。用户可以通过双指缩放和单指拖动操作坐标系视图，便于查看不同尺度的定位细节。

## 传感器融合技术

系统使用名为 SensorFusionManager 的专用类进行航向获取，其核心策略如下：

- 正常情况下，利用**加速度计和磁力计**通过 getRotationMatrix 和 getOrientation 计算手机朝向（方位角）
- 当检测到磁干扰（磁场强度绝对值低于 20μT 或高于 70μT）时，自动切换至**陀螺仪积分**模式，对 Z 轴角速度进行积分以维持航向输出
- 磁干扰消除超过 2 秒后，自动恢复磁力计模式，防止陀螺仪积分误差的累积

这种传感器融合策略有效提高了在复杂电磁环境中航向估计的鲁棒性。

## 信号处理与滤波

系统设计了多层信号处理方案：

- **RssiSlidingWindow**：采用 15 组 FIFO 滑动窗口滤波器，对 RSSI 序列进行均值滤波，至少需要 3 个数据点即可开始计算
- **卡尔曼滤波器**（单点定位模块）：Q = 0.01, R = 2.0，用于实时跟踪 RSSI 变化
- **卡尔曼滤波器**（多点定位模块）：自适应初始化，仅当收到首个测量值后开始滤波，确保冷启动时的准确性
- **距离变化率约束**：限制距离更新速率为每秒不超过 3 米，有效抑制 RSSI 跳变带来的距离估算异常

## 技术栈与开发环境

- 开发语言：Java
- 最低 SDK 版本：Android 8.0（API 26）
- 目标 SDK 版本：Android 14（API 34）
- 构建工具：Gradle 8.3.0
- 编译 SDK：34
- 核心依赖：AndroidX AppCompat 1.6.1、Material Design 1.8.0、Apache Commons Math3 3.6.1
- 权限管理：支持 Android 12+ 的新蓝牙权限模型（BLUETOOTH_SCAN、BLUETOOTH_CONNECT），同时兼容 Android 11 及以下版本的传统蓝牙权限

## 项目结构

```
app/src/main/java/com/ckos/ble/
├── MainActivity.java              // 主活动，管理权限申请、蓝牙初始化和标签页切换
├── Bluetooth.java                 // 蓝牙设备数据模型
├── MyAdapter.java                 // 设备列表自定义适配器
├── RssiSlidingWindow.java         // RSSI滑动窗口滤波器
├── SensorFusionManager.java       // 传感器融合管理器（加速度计+磁力计+陀螺仪）
├── Point.java                     // 坐标点数据模型
└── ui/main/
    ├── FirstFragment.java         // 蓝牙设备搜索页面
    ├── SecondFragment.java        // 单点定位页面
    ├── ThirdFragment.java         // 三点定位导航页面
    ├── SectionsPagerAdapter.java  // 标签页适配器
    ├── CompassView.java           // 罗盘导航自定义视图
    └── CoordinateSystemView.java  // 坐标系可视化自定义视图
```
